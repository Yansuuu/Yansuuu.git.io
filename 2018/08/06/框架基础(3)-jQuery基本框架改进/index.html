改进ie8中apply只能展开真数组和内置伪数组问题

借用数组的slice方法将伪数组转为真数组

[].slice.call(伪数组)

//改进判断是否是数组和伪数组部分
if (jQuery.isLikeArray(selector)) {
    // 如果是数组或者伪数组，每一项分别添加入实例中
    // 将伪数组转为真数组来兼容ie8 
    // ie678中apply只能展开真数组和arguments等内置伪数组
    //将可能是伪数组的selector转为真数组
    [].push.apply(this, [].slice.call(selector));
}

DOMContentLoaded事件

DOMContentLoaded事件：

  • 当DOM元素解析完成时触发
  • 一般情况下比onload快
  • 特殊情况下，与onload差别也不大
  • 所以使用DOMContentLoaded来监听DOM解析
    `javascript
    document.addEventListener(“DOMContentLoaded”, fn);

#### ie8兼容问题

addEventListener 与 DOMContentLoaded都不被ie8支持

替换为 attachEvent 和 onreadystatechange

```javascript
document.attachEvent("onreadystatechange", function () {
        // 需要判断readyState的状态，避免多次调用
        if ( document.readyState === "complete" ) {
            fn();
        }
    });

融入到框架中

添加到jQuery的静态方法中

jQuery.extend({
    ······

    ready: function ( fn ) {
        // 首先判断是否已经简析完成DOM：通过readtState === complete true 直接执行
        // false 然后判断是否支持addEL: 支持绑上DCL
        // 不支持，绑上attachE: 绑上onreadystatechange代替，记得判断readyState避免多次调用

        // 判断DOM是否已经解析完成
        if ( document.readyState === "complete" ) {
            // 已经解析完成 直接调用
            fn();
        } else if ( document.addEventListener ) {
            document.addEventListener("DOMContentLoaded", fn);
        } else {
            document.attachEvent("onreadystatechange", function () {
                // 需要判断readyState的状态，避免多次调用
                if ( document.readyState === "complete" ) {
                    fn();
                }
            });
        }
    }

    ······
});
