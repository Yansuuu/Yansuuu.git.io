匀速动画和缓动动画（简易）封装笔记

Yansuuu 5月 04, 2018
  • javascript
  • 笔记

动画原理

未来位置 = 现在位置 + 步长

再配合定时器每隔第一定时间调用，就可以出现位置变动动画效果

这次的笔记是初次接触js动画封装的记录，只封装了横向运动动画效果

匀速动画简易封装

function animate(ele, target) {
    //注意：要使用定时器前要先清除定时器，这样就阻止了多次调用animate时多个定时器干扰变动速度
    clearInterval(ele.timer);
    //匀速动画酱步长放在定时器外面，因为匀速所以步长无需改变，并且目标位置和现在位置也是固定不变的，所以无需放在定时器中一遍一遍判断
    var speed = target > ele.offsetLeft ? 10:-10;
    //这里讲定时器作为移动元素的一个属性，这样移动多个元素的时候，互相之间的定时器不会干扰，清除自己的定时器也不会清楚到别人的定时器
    ele.timer = setInterval(function () {
        //这里的目标位置和元素现在位置的差值在之后会使用到
        var val = target - ele.offsetLeft;
        //动画原理：未来的位置 = 现在位置 + 步长（注意这里的px）
        ele.style.left = ele.offsetLeft + speed + "px";

        //根据差值的绝对值和步长的绝对值大小来判断是否清除定时器
        //如果差值小于步长了，就没有必要再继续移动一个步长，直接让它到达目标位置
        if (Math.abs(val) < Math.abs(speed)) {
            //清除定时器
            clearInterval(ele.timer);
            //注意这里的px
            ele.style.left = target + "px";
        }
    }, 30)
}

缓动动画简易封装

缓动动画指的是变动速度由快到慢，有一种刹车的效果

function animate(ele, target) {
    //要用定时器先清除定时器
    clearInterval(ele.timer);

    //把定时器绑定到节点的一个属性，这样各个定时器之间互不干扰
    ele.timer = setInterval(function () {
        //缓动动画获取步长是在定时器里面的
        //因为每次步长都不一样，缓动动画算法，步长越来越小
        var step = (target - ele.offsetLeft)/10;
        //对步长二次加工，小于0向下取整，大于零向上取整
        //当目标与现在位置差值小于10的时候，步长就开始为小数了
        //而当步长为0.4的时候，offset获取值就会四舍五入
        //也就是说会在比如396月396.4之间循环，永远无法到达目标
        //所以这里对步长向上向下取整，使得步长可以取到1和-1，在最后10px的时候可以1像素1像素的到达目的位置
        step = step > 0 ? Math.ceil(step) : Math.floor(step);
        //检测是否清除了定时器 如果清除了定时器，它会停止输出
        console.log(1);
        //动画原理，未来位置 = 现在位置 + 步长
        ele.style.left = ele.offsetLeft + step + "px"
        // 设置清除定时器条件 ，要小于等于，因为当差值为0的时候也要清除
        if (Math.abs(target - ele.offsetLeft) <= Math.abs(step)) {
            // 处理小数位置差
            ele.style.left = target + "px"; 
            clearInterval(ele.timer);
        }
    }, 20);
}

两张动画效果对比

匀速动画简单明快，适合轮播图

缓动动画则更加细腻有趣，适用得当的话，比如使用在页面跳转回顶部，有助于提升用户体验
